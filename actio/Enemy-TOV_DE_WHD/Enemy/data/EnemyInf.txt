PROGRAMM-PARAMETER
------------------

Durch Ändern der Startup-sequence können dem Programm Enemy
Parameter übergeben werden. Normalerweise ist dies jedoch
nicht nötig. Folgende Parameter stehen zur Verfügung:

pal    : Erzeugt PAL-Screen (d.h. Bild wird zentriert).
ntsc   : Erzeugt NTSC-Screen.
nofx   : Ladet keine Musik und keine Soundeffekte.
noce   : Macht keine Copperfarbeffekte.
disp   : Lässt das System-Display an.
blnk   : Zeigt Decrunchen durch Blinken an.
wfi    : Zeigt vor jedem Nachladen eine Liste der benötigten
         Files.
wary   : Programm soll vorsichtig sein. Kann benutzt werden,
         wenn Probleme auftreten.
cach   : Lässt die CPU-Caches AN! Diese werden normalerweise
         vom Programm deaktiviert, da sonst bei Amigas mit
         schnellen Prozessoren (z.B. 68060) grafische Fehler
         auftreten können oder der Computer 'abstürzt'.
ints   : Lässt Blitter- und CopperInterrupts zu. Diese werden
         normalerweise abgestellt, da sie nicht benötigt
         werden und zu Schwankungen der Ablaufgeschwindigkeit
         führen können.
noin   : Kopiert beim Installieren auf Festplatte die Files
         nicht wirklich.
hzXX   : Legt MonitorFrequenz fest (XX = Bilder pro Sekunde).


Beispiel:

EF/Enemy >nil: pal hz90

In diesem Falle wird geht das Spiel davon aus, dass es auf
einem PAL-Amiga mit 90 Hertz-Monitor gestartet wurde.

Beim Intro (EnIntro) stehen die Parameter pal, ntsc und cach
zur Verfügung.

Achtung, durch das Installieren auf Festplatte gehen die
Parameter verloren, da die startup-sequence erneuert wird.


INSTALLATION AUF FESTPLATTE
---------------------------

INSTALLATION VON ENEMY AUS
Zum Installieren kann das Enemy-interne Installations-
Programm benutzt werden. Falls nach der Installation die
Startup-sequence der Diskette nicht angepasst wird, kann
dies nachträglich von Hand getan werden.
Dazu benutzt man das File ENEMY_A:S/startup-HD als neue
startup-sequence, wobei man noch ganz am Anfang eine Zeile
hinzufügen muss. In dieser muss das Enemy-Verzeichniss zum
aktuellen Verzeichniss ernannt werden.
Beispiel, wenn die Files nach DH0:ENEMY kopiert wurden:
cd DH0:ENEMY
Diese Zeile muss zuoberst stehen.

Tritt während der Installation ein Fehler auf, wird oft ein
IO-Fehler-Code angegeben. Die wichtigsten sind:

210: KEIN GÜLTIGER DIRECTORY-NAME.
     (z.b. nur dh0: statt "dh0:enemy)
213: DISK NICHT VALIDIERT.
214: DISK SCHREIBGESCHÜTZT.
218: DEVICE NICHT ANGEMELDET (not mounted).
221: DISK IST VOLL.
225: KEINE GÜLTIGE DOS-DISKETTE.
226: KEINE DISKETTE IM LAUFWERK.


INSTALLATION VON DOS AUS
Enemy kann auch 'von Hand' installiert werden:

1) Auf der Festplatte ein Verzeichniss erstellen. Beispiel:

   > MakeDir DH0:ENEMY

2) Von Diskette A die Verzeichnisse 'EF' und 'ENIF' in dieses
   Haupt-Verzeichniss hineinkopieren.

   > Copy ENEMY_A:EF DH0:ENEMY/
   > Copy ENEMY_A:ENIF DH0:ENEMY/

3) Von Diskette B das Verzeichniss 'EF' IN DAS SCHON
   BESTEHENDE Verzeichniss EF auf der Festplatte kopieren.

   > Copy ENEMY_B:EF DH0:ENEMY/

4) Die Verzeichnisse 'EF' und 'ENIF' auf der Festplatte in
   'E' und 'ENI' umbenennen.

   > Rename DH0:ENEMY/EF DH0:ENEMY/E
   > Rename DH0:ENEMY/ENIF DH0:ENEMY/ENI

5) Zum Starten von Enemy das Enemy-Verzeichniss zum aktuellen
   Verzeichniss machen. Das Intro ("Enintro") und das
   Hauptprogramm ("Enemy") starten.

   > CD DH0:ENEMY
   > ENI/Enintro
   > E/Enemy

Hinweis: wenn das Hauptprogramm gestartet wird, sollte kein
CLI-Fenster offen sein, da das Programm sonst blockieren kann
oder grafische Fehler erzeugt werden können.
Lässt sich ein CLI-Fenster nicht vermeiden, so sollte dieses
nach dem Starten mit der Maus sofort ganz klein gemacht und
deaktiviert werden (durch anklicken des leeren Screen-
Bereiches). Der Mauspfeil sollte dann sofort weit vom CLI-
Fenster entfernt positioniert werden.

PROGRAMM LÄUFT NICHT?
---------------------

MITTEILING AN ANACHRONIA
Wenn ENEMY auf Deinem Amiga nicht läuft, teste es wenn möglich
auf einem anderen Amiga.
Wenn die Disketten fehlerhaft sind (Lese-Fehler), so sende sie
an uns zurück.

Läuft das Spiel nicht oder nicht korrekt, obwohl die Disketten
in Ordnung sind, so teile uns dies bitte mit. Bitte mit
Angabe von
1. Amiga-Typ (A500, A1200, A2000b usw).
2. Wieviel RAM-Speicher hat das Gerät und welche Geräte sind
   angeschlossen bzw intern eingebaut.
3. Welcher Prozessor, Grafik-Karten, Turbo-Boards werden benutzt.
4. Möglichst genaue Beschreibung des Problems (an welcher
   Stelle passiert was, bei Gurus: welche Nummer wird
   angezeigt).


ZU WENIG SPEICHER
Beachte: Viele am Amiga angeschlossene Geräte wie z.B.
Festplatten und Disketten-Laufwerke benutzen RAM-Speicher.
Bei zu wenig verfügbarem Speicher wird ENEMY nicht laufen.
Bei Amigas mit 1 MB RAM und einer Festplatte dürfen keine
Disketten-Laufwerke mehr angeschlossen sein! Schalte diese
ab oder hänge sie ab.

Damit mehr Speicher zur Verfügung steht, wird nach dem Intro
der WorkbenchScreen durch den CLI-Befehl CloseWB
geschlossen (Bildschirm wird einfarbig rot).
Falls dieser Befehl auf deinem Computer nicht funktioniert
(Screen bleibt an), so ersetze ihn durch den Befehl OnePlane
und teste, ob dies funktioniert. Einige Rechner stürzen
allerdings durch "OnePlane" ab. Vielleicht hast Du einen anderen
Befehl, welcher den Screen abschaltet oder auf 2 Farben reduziert.


SCHNELLE AMIGAS
Hinweis für Besitzer eines Amigas mit schnellem Prozessor
(z.B. 68060): die Cache-Speicher der CPU müssen deaktiviert
sein. Normalerweise geschieht dies automatisch vom Programm
aus. Falls diese Abschaltung aus irgendeinem Grunde nicht
gelingt, können grafische Fehler oder Abstürze auftreten.
In diesem Falle sollte die Abschaltung manuell im Startup-
Menu des Computers durchgeführt werden (durch Anwählen der
Option "No Caches" bzw. abschalten von "Caches Enabled").


READ-ERROR
Wenn die Diskette B einen Lesefehler aufweist, so kann man
sie vom DiskDoktor bearbeiten lassen. Werden dabei Files als
unreparierbar angezeigt, so lassen sich diese manchmal
ersetzen: auf Diskette A befindet sich ein Directory RESERVE,
in dem sich die Files von den Directorys TXT, SEK und FX der
Diskette B nochmal befinden. Kaputte Files von anderen
Directorys oder von Diskette A können nicht ersetzt werden.


ABSTURZ NACH DEM INTRO
1. Intro statt mit der Maustaste mit dem Joystick-Knopf im
   Port 1 (nicht im Port 0) abbrechen oder mit Taste F1
   Intro ganz vermeiden.
2. Nach dem Intro wird der CLI-Befehl CloseWB aufgerufen.
   Vieleicht erzeugt dieser Befehl auf deinem Rechner einen
   Absturz. Ersetze ihn in diesem Falle durch den Befehl
   OnePlane.


BILD NICHT ZENTRIERT ODER
MUSIK ZU SCHNELL ODER
MONITOR-FREQUENZ AUF 60 STATT AUF 50
Wenn das Programm öfters mal im NTSC-Modus startet, obwohl
Du einen PAL-Amiga hat (Bild ganz am oberen Rand und Musik
läuft zu schnell), so benutze den Parameter pal (siehe oben).


PROGRAMM SOLL VORSICHTIG SEIN
In gewissen Fällen lässt sich ein Problem beheben, wenn
dem Programm der Parameter 'wary' übergeben wird. Die Zeile
in der Startup-Sequence muss dann etwa so lauten:

EF/Enemy >nil: wary
