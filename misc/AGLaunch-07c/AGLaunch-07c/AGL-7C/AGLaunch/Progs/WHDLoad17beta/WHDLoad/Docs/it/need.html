Requisiti di sistema

... per avviare i programmi installati

  • CPU 68000 (è raccomandata una CPU 68010+ perché qualche installatore richiede il VBR per il 'quit by a key' (esci con un tasto))
  • Kickstart 2.0 (versione 37+)
  • un minimo di 1.0 MB RAM (a volte di più, dipende dal programma installato)
Emulazione Kickstart

Qualche installatore utilizza quello che viene chiamato kickemu per ottenere un ambiente AmigaOS completo per il programma installato. Per il kickemu serve l'immagine di un kickstart. Questa immagine può essere estratta da un vero Amiga con programmi quali GrabKick o simili che possono essere trovati su Aminet. La versione supportata dell'immagine dipende dal programma installato. Dovresti poter trovare qualche informazione riguardo alla versione da utilizzare nel file ReadMe allegato all'installatore. L'immagine per le versioni 1.2 e 1.3 del kickstart può essere ottenuta solo da un A500, la versione 3.1 può essere estratta da un A500/A600/A2000,A1200 o un A4000 che naturalmente deve avere la Kick3.1 ROM. Se l'immagine non può essere trovata da WHDLoad oppure è la versione sbagliata o alterata in qualche modo, WHDLoad terminerà l'esecuzione con un apposito messaggio d'errore. In aggiunta a questo, un file di rilocazione con l'estensione .RTB associato all'immagine del kickstart deve essere presente. Questo file di rilocazione può essere trovato su Aminet nell'archivio util/boot/skick346.lha. Entrambi i file, l'immagine del kickstart e il file di rilocazione, devono trovarsi nella directory Devs:Kickstarts/ ed essere nominati nel modo corretto. A partire dalla versione 16.5 di WHDLoad possono essere utilizzate anche le immagini del kickstart presenti nel CD Amiga Forever di Cloanto. Le immagini devono essere rinominate seguendo la tabella qui in basso. In aggiunta rom.key deve essere copiata in Devs:Kickstarts/. Alcuni vecchi installatori hanno bisogno dell'immagine kick40068.A1200, questi programmi non funzioneranno con le ROM Cloanto.

... per scrivere nuovi Slave / installare programmi

  • si raccomanda una CPU 68030+ con MMU per utilizzare tutte le funzioni di WHDLoad (una macchina potente è sempre una scelta azzeccata per sviluppare software ;-))
  • conoscenza del linguaggio macchina, assembly, hardware Amiga ...
  • un assembler (dovrebbe essere in grado di ottimizzare)
  • qualche altro tool può essere utile (editor esadecimali, disassemblatori, ...)
  • qualche Mb aggiuntivo di RAM
  • accesso a differenti configurazioni per il testing (68000 - 68060, OCS, AGA, schede grafiche)
  • un freezer hardware o software come AR3, HrtMon, TK
  • un pulsante NMI per interrompere un programma in esecuzione o per accedere ad un software-monitor/freezer
