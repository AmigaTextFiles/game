FAQ Para a Versão 0.31 Build 650
--------------------------------

*P1) Estou saindo do fuhquake v0.30 para o v0.31 .  Que mudanças eu preciso fazer?
*R1) Há algumas mudanças que você precisa fazer:

     i) Se você usa um charset 24bits, você precisa movê-lo para qw/textures/charsets/XXX.png (ou .tga)
        e colocar "loadcharset XXX" no seu config ler o charset. Você pode muda XXX para o que você quiser.

    ii) Algumas variávies foram renomeadas, então você precisa atualizar seu config:
        gl_picmip_all foi renomeada para gl_scaleModelTextures . O valor padrão é 0.
        scr_sshot_type foi renomeada para sshot_format.
        scr_sshot_dir foi renomeada para sshot_dir.

   iii) $serverstatus foi renomeado para $matchstatus.

*P2) Eu recebo um erro que está faltando "libpng" ou "libjpeg". O que eu faço?
*R2) Se você está usando windows, tenha certeza que você instalou todos os dll's que vem com
     o FuhQuake v0.31.
     Se você está usando linux, você precisa instalar a libpng (>=v1.2.0), libjpeg e libz.
     Todas as distribuições do linux vem com essas bibliotecas e a maioria instala elas como padrão.

*P3) Eu estou usando o linux e tenho certeza que eu tenho o libpng instalado, mas o FuhQuake ainda
     diz que não consegue achar o libpng. O que está errado?
*R3) Algumas distribuições do linux, especialmente o Debian, instala o libpng de uma forma não-padrão.
     Eles instalam o libpng mas eles não criam um link no libpng.so para a verdadeira biblioteca.
     Você precisa criar esse link. Primeiramente ache onde o libpng está instalado ("locate libpng" pode
     se útil). Você vai achar alguma coisa como /usr/lib/libpng.so.X.Y.Z onde X, y e Z são números.
     Então, como root, você faz um "ln -s /usr/lib/libpng.so.X.Y.Z /usr/lib/libpng.so" , isso vai criar
     um link chamado "libpng.so" que aponta para "libpng.so.X.Y.Z" . Se tiver mais de um arquivo
     com o nome no formato "libpng.so.X.Y.Z" então escolhe a versão mais atual (X.Y.Z é a versão).

*P4) Por que cl_camhack, scr_sshot_apply_palette e gl_fb_depthhack sumiram?
*R4) Porque eu os removi.

*P5) O que é esse novo comand, o loadcharset ?
*R5) Nas versões antigas do FuhQuake (v0.30 e anterior), o FuhQuake olhava apenas uma imagem
     chamada charset.png (ou .tga) e se ele encontrasse então ela seria a fonte do seu console.
     A partir do FuhQuake v0.31 você poderá mudar a fonte do seu console através do FuhQuake.
     Coloque os charset's em qw/textures/charset/*.png (ou .tga) e use /loadcharset XXX
     para ler o charset XXX.png (ou .tga). "/loadcharset original" vai ler o charset 8bits
     do seu gfx.wad (isso é o padrão). Note que /loadcharset é só um 'atalho' para a variável
     gl_consolefont .

*P6) Eu ouvi falar que o FuhQuake tem suporte a umas 'degeneration textures'. O que é isso e como eu
     as uso?

*R6) As texturas Degeneration fazem com que o FuhQuake GL fique com a aparência do FuhQuake software com
     d_mipcap 3. Se você não sabe o que é isso então nem se preocupe com isso. O alias abaixo serve para
     ativar esse efeito.

     alias degeneration-2         "gl_lightmode 2; gl_scaleTurbTextures 0; gl_scaleModelTextures 1; gl_texturemode gl_nearest; gl_externalTextures_bmodels 0; gl_externalTextures_world 0; gl_picmip 0; gl_miptexLevel 3; cvar_reset gl_max_size"

     Veja P8 para mais aliases, incluindo um para desabilitar o efeito deegeneration.

*P7) Eu estou louco ou o gl_picmip e gl_max_size ficaram diferentes?
*R7) Elas mudaram um grande detalhe. Elas agora diminuem o tamanho das texturas usando um algorítimo
     muito mais preciso para dar uma maior precisão nas cores. Você não poded voltar a usar o método
     antigo, mas 99% das pessoas vai preferir o novo método.
     Veja http://www.fuhquake.net/forum/viewtopic.php?t=2085 para screenshots.

*P8) Quais são todos os novos comandos de texturas na versão v0.31 ?
     What are all these new texture commands in v0.31 ?
*R8) Todas as variávies abiaxo nescessitam que você reinicie o mapa para elas terem efeito.

     i) gl_miptexLevel padrão é 0 e pode ter os valores 0, 1, 2 ou 3. É escencialmente
        um equivalent ao d_mipcap no GL. Isso não tem efeito quando usando texturas externas
	    em 24bits.

    ii) gl_externalTextures_bmodels poded ser setado para 0 para não ler texturas 24bits externas
        para .bsp que não sejam mapas (ex.: models de health e de munição).
        gl_externalTextures_world pode ser usado para desabilitar a leitura de texturas externas
	    24bits para os .bso que são realmente mapas.

   iii) gl_scaleModelTextures determina se gl_picmip/gl_max_size/gl_miptexLevel afeta models
        diferentes do mapa proriamente dito. A maioria das pessoas vai usar isso com o valor 0
	    (que é o padrão). Essa variável era chamada de gl_picmip_all .

    iv) gl_scaleTurbTextures determin se gl_picmip/gl_max_size/gl_miptexLevel afeta texturas
        líquidas (lava, água, slime, teleports).

     Aqui tem algumas demonstrações das nova variáveis:

     alias degeneration-1	"gl_lightmode 2; gl_scaleTurbTextures 0; gl_scaleModelTextures 1; gl_texturemode gl_nearest; gl_externalTextures_bmodels 0; gl_externalTextures_world 0; gl_picmip 3; gl_miptexLevel 0; cvar_reset gl_max_size"
     alias degeneration-2	"gl_lightmode 2; gl_scaleTurbTextures 0; gl_scaleModelTextures 1; gl_texturemode gl_nearest; gl_externalTextures_bmodels 0; gl_externalTextures_world 0; gl_picmip 0; gl_miptexLevel 3; cvar_reset gl_max_size"
     alias degeneration-3	"gl_lightmode 2; gl_scaleTurbTextures 0; gl_scaleModelTextures 0; gl_texturemode gl_linear_mipmap_nearest; gl_externalTextures_bmodels 1; gl_externalTextures_world 0; gl_picmip 0; gl_miptexLevel 0; gl_max_size 1"
     alias regeneration-1	"gl_lightmode 0; gl_scaleTurbTextures 0; gl_scaleModelTextures 0; gl_texturemode gl_linear_mipmap_nearest; gl_externalTextures_bmodels 1; gl_externalTextures_world  1; gl_picmip 0; gl_miptexLevel 0; cvar_reset gl_max_size"

     As 3 primeiras desabilitam texturas 24bits, e aplicam uma das variávies, gl_picmip, gl_miptexLevel
     ou gl_max_size. O Quarto alias restaura os antigos efeitos.
     Você pode querer usar gl_linear_mipmap_linear ao invéz de gl_linear_mipmap_nearest nos 2 últimos
     aliases para obter uma qualidade de imagem melhor, isso ao custo de algums FPS.

*P9) Como eu habilito o uso de screenshots em PNG no FuhQuake Software?
*R9) Use "sshot_format png" para tirar screenshots em png. Altere "image_png_compression_level"
     para ajustar a compressão (9 é o máximo de compressão, 0 não comprime).
